Background by Anthony Zhang

Must Haves

  • Fit large amounts of data

    The table should make data easier for users to digest.

  • Responsive & Accessible

    Standard web experience stuff.
    Explore different layout options for mobile / smaller screens.

  • Defined use cases

    Make sure the user has a proper use case for implementing a table.
    If not, offer alternative UI that best fits their needs.

When to use Data Tables?

  • User Comfort

    The user is comfortable with a grid style (i.e., excel spreadsheets)

  • Edit/Modify Values

    User needs to find & modify existing data

  • Analyzing granular data

    For summaries and/or overviews, use graphs
